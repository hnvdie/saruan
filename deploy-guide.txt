  ---- --------------------------------------------------------------
   01  Deploy Awal ke VPS

  ---- --------------------------------------------------------------

Spesifikasi VPS

  -------------------- ----------------------- -----------------------
  Komponen             Minimum                 Rekomendasi

  CPU                  1 vCPU                  2 vCPU

  RAM                  512 MB                  1-2 GB

  Storage              10 GB SSD               20 GB SSD

  OS                   Ubuntu 22.04            Ubuntu 22.04 LTS

  Provider             Contabo / DO            Hetzner CX21
  -------------------- ----------------------- -----------------------

Langkah Setup Server

  -------------------------------------------------------------------
  # 1. Login & update OS

  ssh root@IP_VPS_KAMU

  apt update && apt upgrade -y

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # 2. Buat user non-root (jangan deploy dari root!)

  adduser deploy

  usermod -aG sudo deploy

  su - deploy

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # 3. Install dependencies

  sudo apt install -y python3 python3-pip python3-venv nginx git curl
  ufw

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # 4. Firewall — WAJIB jangan skip

  sudo ufw allow OpenSSH

  sudo ufw allow "Nginx Full"

  sudo ufw enable

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # 5. Clone repo private dari GitHub

  cd /var/www

  sudo mkdir undangankita && sudo chown deploy:deploy undangankita

  # Pakai HTTPS + Personal Access Token (PAT):

  git clone https://TOKEN@github.com/USERNAME/REPO.git undangankita

  # Atau pakai SSH deploy key (lihat Section 05)

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # 6. Setup virtualenv & install packages

  cd /var/www/undangankita

  python3 -m venv venv

  source venv/bin/activate

  pip install -r requirements.txt gunicorn

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # 7. Buat .env di VPS (isi ulang manual, jangan copy dari HP)

  nano /var/www/undangankita/.env

  chmod 600 /var/www/undangankita/.env

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # 8. Init database

  python -c "from app import init_db; init_db()"

  mkdir -p /var/log/undangankita

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # 9. Install systemd service

  sudo cp habarkita.service /etc/systemd/system/

  sudo systemctl daemon-reload

  sudo systemctl enable undangankita

  sudo systemctl start undangankita

  sudo systemctl status undangankita # harus: active (running)

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # 10. Setup Nginx

  sudo cp nginx.conf /etc/nginx/sites-available/undangankita

  sudo ln -s /etc/nginx/sites-available/undangankita
  /etc/nginx/sites-enabled/

  sudo nginx -t && sudo systemctl reload nginx

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # 11. SSL gratis dengan Certbot

  sudo apt install -y certbot python3-certbot-nginx

  sudo certbot --nginx -d habarkita.com -d www.habarkita.com

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  Verifikasi deploy: curl -I https://habarkita.com → harus 200, ada
  header Strict-Transport-Security, tidak ada versi Nginx.

  ---- --------------------------------------------------------------
   02  Update Kode & Maintenance

  ---- --------------------------------------------------------------

Alur Deploy Update (dari Termux ke VPS)

+--------------------------------+---------------------------------+
| Di Termux (HP)                 | Di VPS                          |
+--------------------------------+---------------------------------+
| git add -A                     | cd /var/www/undangankita        |
|                                |                                 |
| git commit -m "pesan update"   | git pull origin main            |
|                                |                                 |
| git push origin main           | source venv/bin/activate        |
|                                |                                 |
| ⚠ Pastikan .env tidak ikut     | pip install -r requirements.txt |
| push!                          |                                 |
|                                | sudo systemctl restart          |
|                                | undangankita                    |
+--------------------------------+---------------------------------+

Script deploy.sh (satu command untuk semua)

  -------------------------------------------------------------------
  #!/bin/bash

  # /var/www/undangankita/deploy.sh

  set -e

  echo "Pulling code..."

  git pull origin main

  source venv/bin/activate && pip install -r requirements.txt -q

  sudo systemctl restart undangankita

  echo "Deploy selesai!"

  # Cara pakai:

  bash deploy.sh
  -------------------------------------------------------------------

Mode Maintenance — Halaman Under Maintenance

  -------------------------------------------------------------------
  # Tambahkan di nginx.conf server block (sebelum location /):

  # if (-f /var/www/html/maintenance.on) { return 503; }

  # error_page 503 /maintenance.html;

  # location = /maintenance.html { root /var/www/html; internal; }

  # AKTIFKAN maintenance:

  touch /var/www/html/maintenance.on && sudo nginx -s reload

  # NONAKTIFKAN maintenance:

  rm /var/www/html/maintenance.on && sudo nginx -s reload
  -------------------------------------------------------------------

  -------------------------------------------------------------------
  Dengan cara ini Nginx tetap running dan serve halaman maintenance.
  Flask bisa di-restart bebas tanpa visitor tahu.

  -------------------------------------------------------------------

Monitoring

  -------------------------------------------------------------------
  sudo systemctl status undangankita # status app

  sudo journalctl -u undangankita -f # log live

  sudo tail -f /var/log/undangankita/error.log # error log

  sudo tail -f /var/log/nginx/access.log # akses log

  ---- --------------------------------------------------------------
   03  Backup Rutin

  ---- --------------------------------------------------------------

Yang perlu di-backup: database (.db) dan folder uploads. Kode aman di
GitHub.

Script Backup Otomatis

  -------------------------------------------------------------------
  #!/bin/bash

  # /home/deploy/backup.sh

  BACKUP_DIR="/home/deploy/backups"

  APP_DIR="/var/www/undangankita"

  DATE=20260308_1930

  mkdir -p

  cp /data/undangan.db /db_.db

  tar -czf /uploads_.tar.gz /static/uploads/

  find -name "*.db" -mtime +30 -delete

  find -name "*.tar.gz" -mtime +30 -delete

  echo "Backup selesai: "
  -------------------------------------------------------------------

  -------------------------------------------------------------------
  chmod +x /home/deploy/backup.sh

  # Jadwal cron — backup setiap hari jam 02:00

  crontab -e

  # Tambahkan:

  0 2 * * * /home/deploy/backup.sh >> /home/deploy/backup.log 2>&1
  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # Pakai rclone — sync ke Backblaze B2 / Google Drive / dll

  -------------------------------------------------------------------

  ---- --------------------------------------------------------------
   04  Disaster Recovery — VPS Mati / Ganti Server

  ---- --------------------------------------------------------------

  -------------------------------------------------------------------
  Skenario: VPS lama mati total, tidak bisa diakses. Harus pindah ke
  VPS baru secepat mungkin.

  -------------------------------------------------------------------

Prosedur Recovery ke VPS Baru

  -------------------------------------------------------------------
  # Step 1: Order VPS baru (Ubuntu 22.04)

  # Bisa dilakukan paralel sementara VPS lama down

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # Step 2: Update DNS ke IP baru

  # Buka panel domain (Cloudflare/Niagahoster)

  # Ubah A record habarkita.com → IP VPS baru

  # Cloudflare: propagasi ~5 menit | DNS biasa: hingga 24 jam

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # Step 3: Setup VPS baru — ikuti semua langkah Section 01

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # Step 4: Restore database dari backup

  rclone copy b2:nama-bucket/db_YYYYMMDD.db /tmp/

  cp /tmp/db_YYYYMMDD.db /var/www/undangankita/data/undangan.db

  chown deploy:deploy /var/www/undangankita/data/undangan.db

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # Step 5: Restore uploads

  rclone copy b2:nama-bucket/uploads_YYYYMMDD.tar.gz /tmp/

  tar -xzf /tmp/uploads_YYYYMMDD.tar.gz -C /

  -------------------------------------------------------------------

  -------------------------------------------------------------------
  # Step 6: Restart & verifikasi

  sudo systemctl restart undangankita

  curl -I https://habarkita.com # harus 200 OK

  -------------------------------------------------------------------

Tips Meminimalkan Downtime

- Pakai Cloudflare sebagai proxy DNS — propagasi DNS hanya 5 menit saat
  ganti server

- Aktifkan snapshot VPS mingguan di panel provider (~Rp 15rb/bln, worth
  it)

- Catat di HP: IP VPS, username deploy, lokasi backup — supaya kalau
  panik masih ingat

- Selalu jalankan backup.sh manual sebelum maintenance besar

- Backup ke minimal 2 tempat berbeda (contoh: VPS + Backblaze B2)

   05  GitHub Private Repo — Setup Deploy Key

  ---- --------------------------------------------------------------

Supaya VPS bisa git pull tanpa ketik password/token setiap kali:

  -------------------------------------------------------------------
  # Di VPS — buat SSH key khusus untuk deploy

  ssh-keygen -t ed25519 -C "vps-deploy" -f ~/.ssh/github_deploy -N ""

  cat ~/.ssh/github_deploy.pub # copy outputnya

  # Di GitHub:

  # Repo → Settings → Deploy Keys → Add deploy key

  # Paste public key, nama: VPS Deploy, Allow write: TIDAK
  (read-only)

  # Konfigurasi SSH di VPS

  cat >> ~/.ssh/config << EOF

  Host github.com

  IdentityFile ~/.ssh/github_deploy

  StrictHostKeyChecking no

  EOF

  # Ubah remote ke SSH

  cd /var/www/undangankita

  git remote set-url origin git@github.com:USERNAME/REPO.git

  # Test

  git pull # harus jalan tanpa minta password
  -------------------------------------------------------------------

  -------------------------------------------------------------------
  Deploy key bersifat read-only dan hanya berlaku untuk satu repo.
  Jauh lebih aman dari Personal Access Token yang bisa akses semua
  repo.

  -------------------------------------------------------------------

Cek .env Tidak Pernah Ter-commit

  -------------------------------------------------------------------
  # Cek apakah file sensitif sudah terlanjur di-track git:

  git ls-files .env data/ static/uploads/

  # Kalau ada output → WAJIB dihapus dari tracking!

  # Cara hapus (tidak menghapus file aslinya):

  git rm --cached .env

  git rm --cached -r data/

  git commit -m "remove sensitive files"

  git push
  -------------------------------------------------------------------

  -------------------------------------------------------------------
  Kalau .env sudah ter-push ke GitHub: anggap SECRET_KEY &
  ADMIN_PW_HASH bocor. Generate ulang keduanya dan update .env di VPS
  segera.
